## โครงการ AI Smart Re-Triage Assistant

## Data Dict ของชุดข้อมูล
- **patient_id**: รหัสประจำตัวของผู้ป่วยแต่ละราย
- **age**: อายุของผู้ป่วย
- **initial_triage_level**: ระดับความเร่งด่วนจากการคัดกรองครั้งแรก
- **systolic_bp**: ค่าความดันโลหิตตัวบน
- **diastolic_bp**: ค่าความดันโลหิตตัวล่าง
- **pulse**: อัตราการเต้นของชีพจร
- **rr**: อัตราการหายใจ
- **spo2**: ค่าความอิ่มตัวของออกซิเจนในเลือด
- **temp**: อุณหภูมิร่างกาย
- **wait_minutes**: ระยะเวลารอหลังการคัดกรองครั้งแรก
- **repeated_measure_count**: จำนวนครั้งที่มีการวัดซ้ำ
- **delta_pulse**: ค่าการเปลี่ยนแปลงของชีพจรจากครั้งก่อน
- **delta_bp**: ค่าการเปลี่ยนแปลงของความดันโลหิตจากครั้งก่อน
- **label**: ป้ายกำกับผลลัพธ์ของผู้ป่วยในเชิงการตอบสนอง ได้แก่ `continue_waiting`, `observe_closely` และ `retriage_now` ใช้เป็นตัวแปรเป้าหมายในการวิเคราะห์แบบ classification

## 1) แนวคิดในการจัดการข้อมูล Data Cleansing

ประเด็นสำคัญในการทำ data cleansing สำหรับชุดข้อมูลนี้ ได้แก่

- ตรวจสอบชนิดข้อมูลของแต่ละคอลัมน์ให้สอดคล้องกับความหมายของตัวแปร เช่น อายุควรเป็นจำนวนเต็ม ระดับ triage ควรเป็นจำนวนเต็ม และค่าทางสรีรวิทยาควรอยู่ในรูปตัวเลข โดยสามารถตรวจสอบเบื้องต้นด้วยคำสั่ง `df.info()` และปรับชนิดข้อมูลได้ด้วย `astype()`
- ตรวจสอบค่าว่างหรือ missing values เพื่อดูว่ามีข้อมูลที่หายไปในคอลัมน์ใดบ้าง โดยสามารถใช้คำสั่ง `df.isnull().sum()`
- ตรวจสอบค่าซ้ำซ้อนของระเบียน เพื่อป้องกันไม่ให้ข้อมูลบางรายการถูกนับซ้ำ โดยสามารถใช้คำสั่ง `df.duplicated().sum()`
- ตรวจสอบช่วงค่าที่สมเหตุสมผลของตัวแปร เช่น ค่า SpO2 ควรอยู่ในช่วง 0–100, ค่า `wait_minutes` ไม่ควรติดลบ, และ `temperature` ควรอยู่ในช่วงที่เหมาะสมกับร่างกายมนุษย์ โดยสามารถใช้การกรองข้อมูลด้วยเงื่อนไข เช่น `(df["spo2"] < 0) | (df["spo2"] > 100)` หรือ `(df["wait_minutes"] < 0)`
- ตรวจสอบความถูกต้องของตัวแปร `label` โดยควรมีเฉพาะกลุ่มที่กำหนดไว้ ได้แก่ `continue_waiting`, `observe_closely` และ `retriage_now` ซึ่งสามารถตรวจสอบได้ด้วย `df["label"].value_counts()` หรือ `df[~df["label"].isin([...])]`
- ตรวจสอบ outlier หรือค่าผิดปกติที่อาจส่งผลต่อผลลัพธ์ของโมเดล เช่น ค่าความดันที่ต่ำหรือสูงเกินจริง หรือค่าการเปลี่ยนแปลงของชีพจรที่รุนแรงผิดปกติ โดยสามารถใช้การหาควอไทล์ด้วย `quantile()` และคำนวณ IQR เพื่อระบุค่าที่อยู่นอกช่วงปกติ

### Python libraries ที่ใช้ในการทำ Data Cleansing เบื้องต้น
- **pandas**: ใช้สำหรับอ่านไฟล์ข้อมูล ตรวจสอบ missing values, duplicate rows, ชนิดข้อมูลของแต่ละคอลัมน์ และสรุปข้อมูลเบื้องต้น
- **numpy**: ใช้ช่วยจัดการข้อมูลเชิงตัวเลข และสนับสนุนการคำนวณบางส่วนที่เกี่ยวข้องกับการตรวจสอบข้อมูล
- **pandas describe() / quantile()**: ใช้ตรวจสอบการกระจายของข้อมูลและช่วยหา outlier เบื้องต้น เช่น วิธี IQR
- **เงื่อนไขเชิงตรรกะใน Python**: ใช้ตรวจสอบช่วงค่าที่สมเหตุสมผลของตัวแปร เช่น SpO2, wait_minutes, temperature และ label

ตัวอย่างไฟล์สำหรับการทำ data cleansing คือ **01.ipynb**

## 2) การใช้ Linear Regression ในการวิเคราะห์ข้อมูล

### ตัวอย่างโจทย์ที่ใช้กับ Linear Regression
**“จากข้อมูลผู้ป่วยและสัญญาณชีพเบื้องต้น สามารถพยากรณ์ได้หรือไม่ว่าผู้ป่วยรายหนึ่งจะต้องรอประมาณกี่นาที (`wait_minutes`)”**  

- กำหนดตัวแปรตามให้อยู่ในรูปค่าตัวเลขต่อเนื่อง โดยในกรณีนี้เลือกใช้ `wait_minutes` เป็นตัวแปรตาม เพื่อศึกษาว่าข้อมูลผู้ป่วยและสัญญาณชีพสามารถอธิบายหรือพยากรณ์เวลารอคอยได้มากน้อยเพียงใด โดยสามารถกำหนดตัวแปรอิสระและตัวแปรตามได้ด้วย `X = df[feature_cols]` และ `y = df["wait_minutes"]`
- เลือกตัวแปรอิสระที่เกี่ยวข้องกับบริบทของปัญหา เช่น `age`, `initial_triage_level`, `systolic_bp`, `diastolic_bp`, `pulse`, `rr`, `spo2`, `temp`, `repeated_measure_count`, `delta_pulse` และ `delta_bp` เพื่อใช้เป็นข้อมูลนำเข้าในการสร้างโมเดล
- แบ่งข้อมูลออกเป็นชุดฝึกและชุดทดสอบ เพื่อใช้ประเมินความสามารถของโมเดลในการพยากรณ์ข้อมูลใหม่ โดยสามารถใช้คำสั่ง `train_test_split()`
- สร้างและฝึกโมเดล Linear Regression เพื่อเรียนรู้ความสัมพันธ์เชิงเส้นระหว่างตัวแปรอิสระกับเวลารอคอย โดยสามารถใช้ `LinearRegression()` และ `fit()`
- ใช้โมเดลที่ฝึกแล้วในการพยากรณ์ค่าของ `wait_minutes` จากข้อมูลชุดทดสอบ โดยสามารถใช้คำสั่ง `predict()`
- ประเมินผลของโมเดลด้วยตัวชี้วัดที่เหมาะสม เช่น ค่า MAE, MSE, RMSE และ R² เพื่อดูว่าโมเดลสามารถพยากรณ์ได้ใกล้เคียงกับค่าจริงมากน้อยเพียงใด โดยสามารถใช้ `mean_absolute_error()`, `mean_squared_error()` และ `r2_score()`
- วิเคราะห์ค่าสัมประสิทธิ์ของโมเดล เพื่อดูว่าตัวแปรใดมีแนวโน้มสัมพันธ์กับเวลารอคอยในทิศทางบวกหรือลบ โดยสามารถดูได้จาก `model.coef_` และ `model.intercept_`

### Python libraries ที่ใช้ในการทำ Linear Regression เบื้องต้น
- **pandas**: ใช้สำหรับอ่านไฟล์ข้อมูล เลือกตัวแปรอิสระและตัวแปรตาม และจัดการข้อมูลในรูป DataFrame
- **numpy**: ใช้ช่วยในการจัดการข้อมูลเชิงตัวเลข และช่วยเรียงลำดับหรือแสดงผลค่าสัมประสิทธิ์ของโมเดล
- **scikit-learn.model_selection**: ใช้สำหรับแบ่งข้อมูลเป็น training set และ test set ด้วย `train_test_split()`
- **scikit-learn.linear_model**: ใช้สำหรับสร้างโมเดล Linear Regression ด้วย `LinearRegression()`
- **scikit-learn.metrics**: ใช้สำหรับประเมินผลโมเดล เช่น `mean_absolute_error()`, `mean_squared_error()` และ `r2_score()`

ตัวอย่างไฟล์สำหรับการทำ Linear Regression คือ **02.ipynb**

## 3) การใช้ K-NN ในการวิเคราะห์ข้อมูล

### ตัวอย่างโจทย์ที่ใช้กับ K-NN
**“จากข้อมูลผู้ป่วยและสัญญาณชีพเบื้องต้น สามารถจำแนกได้หรือไม่ว่าผู้ป่วยรายหนึ่งควรถูกจัดอยู่ในกลุ่ม `continue_waiting`, `observe_closely` หรือ `retriage_now`”**

- กำหนดตัวแปรตามให้อยู่ในรูปกลุ่มหรือ class โดยในกรณีนี้เลือกใช้ `label` เป็นตัวแปรตาม เพื่อใช้จำแนกผู้ป่วยออกเป็น 3 กลุ่ม ได้แก่ `continue_waiting`, `observe_closely` และ `retriage_now` โดยสามารถกำหนดตัวแปรอิสระและตัวแปรตามได้ด้วย `X = df[feature_cols]` และ `y = df["label"]`
- เลือกตัวแปรอิสระที่เกี่ยวข้องกับบริบทของปัญหา เช่น `age`, `initial_triage_level`, `systolic_bp`, `diastolic_bp`, `pulse`, `rr`, `spo2`, `temp`, `wait_minutes`, `repeated_measure_count`, `delta_pulse` และ `delta_bp` เพื่อใช้เป็นข้อมูลนำเข้าในการจำแนกกลุ่มผู้ป่วย
- แบ่งข้อมูลออกเป็นชุดฝึกและชุดทดสอบ เพื่อใช้ประเมินความสามารถของโมเดลในการจำแนกข้อมูลใหม่ โดยสามารถใช้คำสั่ง `train_test_split()`
- ปรับขนาดข้อมูลก่อนสร้างโมเดล เนื่องจาก K-NN อาศัยการคำนวณระยะห่างระหว่างข้อมูล หากตัวแปรมีหน่วยหรือช่วงค่าต่างกันมาก อาจทำให้ผลการจำแนกคลาดเคลื่อนได้ โดยสามารถใช้ `StandardScaler()`
- สร้างและฝึกโมเดล K-NN เพื่อเรียนรู้ลักษณะของข้อมูลที่อยู่ใกล้กัน และใช้เพื่อนบ้านที่ใกล้ที่สุดในการตัดสินว่าข้อมูลใหม่ควรถูกจัดอยู่ในกลุ่มใด โดยสามารถใช้ `KNeighborsClassifier()` และ `fit()`
- ใช้โมเดลที่ฝึกแล้วในการพยากรณ์กลุ่มของผู้ป่วยจากข้อมูลชุดทดสอบ โดยสามารถใช้คำสั่ง `predict()`
- ประเมินผลของโมเดลด้วยตัวชี้วัดที่เหมาะสม เช่น Accuracy, Precision, Recall, F1-score และ Confusion Matrix เพื่อดูว่าโมเดลสามารถจำแนกผู้ป่วยได้ดีมากน้อยเพียงใด โดยสามารถใช้ `accuracy_score()`, `classification_report()` และ `confusion_matrix()`

### Python libraries ที่ใช้ในการทำ K-NN เบื้องต้น
- **pandas**: ใช้สำหรับอ่านไฟล์ข้อมูล เลือกตัวแปรอิสระและตัวแปรตาม และจัดการข้อมูลในรูป DataFrame
- **scikit-learn.model_selection**: ใช้สำหรับแบ่งข้อมูลเป็น training set และ test set ด้วย `train_test_split()`
- **scikit-learn.preprocessing**: ใช้สำหรับปรับขนาดข้อมูลด้วย `StandardScaler()` ก่อนนำเข้าโมเดล K-NN
- **scikit-learn.neighbors**: ใช้สำหรับสร้างโมเดล K-NN ด้วย `KNeighborsClassifier()`
- **scikit-learn.metrics**: ใช้สำหรับประเมินผลโมเดล เช่น `accuracy_score()`, `classification_report()` และ `confusion_matrix()`

## 4) แนวทางการประยุกต์ใช้ K-means ในโครงการและการทดลองเบื้องต้น

### ตัวอย่างโจทย์ที่ใช้กับ K-means
**“จากข้อมูลสัญญาณชีพและเวลารอคอย สามารถจัดกลุ่มผู้ป่วยออกเป็นกลุ่มลักษณะใกล้เคียงกันได้หรือไม่ เช่น กลุ่มเสี่ยงต่ำ กลุ่มเฝ้าระวัง และกลุ่มเสี่ยงสูง”**

- กำหนดตัวแปรที่ใช้ในการจัดกลุ่มให้อยู่ในรูปข้อมูลเชิงตัวเลข โดยในกรณีนี้สามารถเลือกใช้ตัวแปร เช่น `systolic_bp`, `pulse`, `spo2`, `temp` และ `wait_minutes` เพื่อใช้สะท้อนลักษณะทางสรีรวิทยาและสถานะการรอของผู้ป่วย
- ตัดตัวแปร `label` ออกก่อนการวิเคราะห์ เนื่องจาก K-means เป็นวิธีแบบ unsupervised learning ที่ไม่ได้ใช้ป้ายกำกับคำตอบล่วงหน้า
- เลือกเฉพาะตัวแปรที่มีความหมายต่อการจัดกลุ่มผู้ป่วย เพื่อให้แต่ละ cluster สะท้อนรูปแบบของผู้ป่วยได้ชัดเจนมากขึ้น
- ปรับขนาดข้อมูลก่อนการจัดกลุ่ม เนื่องจากตัวแปรแต่ละตัวมีหน่วยและช่วงค่าต่างกัน เช่น `pulse`, `spo2` และ `wait_minutes` จึงควรใช้ `StandardScaler()` เพื่อให้ทุกตัวแปรมีอิทธิพลต่อการคำนวณใกล้เคียงกัน
- สร้างโมเดล K-means โดยกำหนดจำนวนกลุ่มเริ่มต้น เช่น `n_clusters=3` เพื่อทดลองแบ่งผู้ป่วยออกเป็น 3 กลุ่มหลัก โดยสามารถใช้ `KMeans()` และ `fit_predict()`
- วิเคราะห์ผลลัพธ์ของแต่ละ cluster โดยดูจำนวนสมาชิกในแต่ละกลุ่ม และดูค่า centroid ของแต่ละกลุ่ม เพื่อช่วยตีความว่ากลุ่มนั้นมีลักษณะเป็นกลุ่มผู้ป่วยแบบใด
- ประเมินคุณภาพเบื้องต้นของการจัดกลุ่มด้วยตัวชี้วัด เช่น Silhouette Score เพื่อดูว่าการแบ่งกลุ่มมีความเหมาะสมมากน้อยเพียงใด โดยสามารถใช้ `silhouette_score()`
- นำผลการจัดกลุ่มไปประยุกต์ใช้ในบริบทของโครงการ เช่น ใช้เพื่อสำรวจว่าผู้ป่วยในพื้นที่รอ ER มีรูปแบบการกระจายตัวอย่างไร หรือใช้เป็นแนวทางในการออกแบบกลุ่มความเสี่ยงเบื้องต้นก่อนสร้างระบบแจ้งเตือนหรือ decision support

### Python libraries ที่ใช้ในการทำ K-means เบื้องต้น
- **pandas**: ใช้สำหรับอ่านไฟล์ข้อมูล เลือกตัวแปรที่ต้องการใช้ในการจัดกลุ่ม และจัดการข้อมูลในรูป DataFrame
- **scikit-learn.preprocessing**: ใช้สำหรับปรับขนาดข้อมูลด้วย `StandardScaler()` ก่อนนำเข้าโมเดล K-means
- **scikit-learn.cluster**: ใช้สำหรับสร้างโมเดล K-means ด้วย `KMeans()`
- **scikit-learn.metrics**: ใช้สำหรับประเมินผลเบื้องต้นของการจัดกลุ่ม เช่น `silhouette_score()`
- **numpy**: ใช้ช่วยจัดการข้อมูลเชิงตัวเลขและสนับสนุนการแสดงผล centroid หรือผลลัพธ์ของ cluster
